一、项目概述

项目背景

现代用户在择偶与亲密关系中常见困惑：

不清楚自己在亲密关系中的真实需求；

只会用“上进、温柔、三观合”等模糊标签选人；

常常“凭感觉谈恋爱”，结果踩坑后才发现“根本不适合”。

本项目希望通过一个系统化的在线测评，帮助用户：看清自己的关系需求 → 匹配适合的伴侣类型 → 提醒应避免的伴侣类型。

产品定位

定位为：“伴侣适配度 / 择偶方向”认知类测评产品，兼具趣味性与实用性。

作为用户进入情感/测评矩阵的入口产品，可用于后续：

延伸更多测评；

打通后续信息产品/咨询服务。

品牌理念："适配恋人"致力于帮助用户先了解自己的情感需求，再找到真正适合的伴侣类型，让爱情不再是试错的过程，而是明智的选择。

项目目标

帮助用户回答三个核心问题：

我在亲密关系中真正需要的是什么？

我更适合什么类型的伴侣？

哪种类型的人，再心动也最好避开？

对产品方：

打造一个可长期使用的标准化测评；

提供可二次开发的维度模型和结果模板。

二、目标用户与使用场景

目标用户画像

年龄：18–40 岁，以 20–35 岁为主；

状态：

单身、在恋爱中、或考虑结婚的人；

对亲密关系有一定反思意愿；

特点：

使用社交媒体、小红书等平台；

对心理测评、人格测试有兴趣。

典型使用场景

单身用户：想搞清自己该找什么样的人，不再“见一个喜欢一个”。

正在恋爱：想知道当前伴侣和自己“适配度如何”、是否踩某些典型雷。

闺蜜/情侣之间：互相做测试，对比结果当话题。

内容创作者：作为分享链接、引流工具。

三、测评整体设计（模型概述）

测评题量与形式

共 46 道题目（含6道反向计分题）；

均为单选题、4 选 1；

题目以具体情境 + 行为/反应选项为主，避免直接自我评价量表。

评估维度（后台使用，不对用户直接公开）

S：情感安全感需求（Security）

A：自主 / 空间需求（Autonomy）

G：共同成长 / 事业重视度（Growth）

R：现实务实度（Reality）

E：情绪共鸣 / 沟通方式（Emotion）

伴侣类型模型（输出对象）

预设 8 种“推荐伴侣类型”（对用户输出）：

稳定港湾型

并肩打拼型

滋养照顾型

灵魂对谈型

自由同伴型

理性拍档型

现实靠谱型

情绪共振型

测评输出形式

固定输出两类结果：

1 个主推荐伴侣类型：与用户需求最匹配；

1 个避雷伴侣类型：与用户需求冲突最大的类型。

同时给出：

用户在 5 个维度上的需求画像（高/中/低 + 简要解释）；

主类型详细说明 + 为什么适合；

避雷类型说明 + 为什么不适合；

若干条“识别信号”与实战建议。

五、测评逻辑与规则（概要）

详细题目与打分逻辑见：《题目 + 评分逻辑（Markdown）》文件

题目结构

共 46 题（含6道反向计分题），每题 4 个选项（A/B/C/D）；

每个选项对应若干维度的加分/减分（例如 S+2, A+1，反向题可能包含负分如 S-2）。

维度得分计算

初始：S = 0, A = 0, G = 0, R = 0, E = 0；

用户完成所有46题后，将每题选项对应的得分累加（包含反向题的负分）；

得到五个维度的原始总分；

做线性映射，转换为 0–100 分制，并确定级别（极高90+、较高70-89、中等40-69、较低20-39、很低<20）。

伴侣类型“理想配置”

每个伴侣类型，预先定义一组“理想维度值”（例如用 0–100 表示）；

示例（仅为逻辑说明，具体数值可在实现中微调）：

稳定港湾型：S 高、R 中偏高、G 中、E 中；

并肩打拼型：G 高、R 高，S/E 中等；

自由同伴型：A 高，S 中，E/R 中等；

……（共 8 类）。

主类型计算逻辑

对每一个伴侣类型：

计算“用户五维得分”与“该类型理想配置”的差值；

差值越小 → 越匹配；

主推荐类型 = 差值最小的那一个类型。

避雷类型计算逻辑

同样使用差值；

避雷类型 = 差值最大的那个类型。

适配度计算
- 主推荐类型适配度 = (100 - 最小差值 × 系数)%
- 其他类型适配度按差值递减
- 适配度用于在报告中直观显示匹配程度

边界情况处理

如出现并列（差值相同）：

优先选择名称更直观、文案更成熟的类型作为主类型；

其他作为内部“备选类型”（首版前端不展示）。

六、UI / 交互与文案注意事项

语气与态度

不做“好坏评判”，只做“适合 vs 不适合”的提示；

避免使用“你有问题”“你很糟糕”等否定性语句；

对高敏感、高需求用户保持友好和支持的语气。

题目文案

尽量使用真实生活场景描述，而不是抽象心理学名词；

每题选项要体现不同风格的“真实反应”，无明显价值判断高低。

结果文案

给用户“被理解”的感觉，而不是被批判；

强调每种类型都有优点与风险，不塑造完美伴侣幻想；

强调：测评结果是“择偶方向参考”，不是绝对结论。

七、非功能需求（简述）

性能

单页面测评流程简洁，加载速度快；

适配移动端访问。

兼容性

主流浏览器（Chrome、Safari、手机端浏览器）正常使用；

无需登录，无需注册。

隐私与安全

不要求用户填写真实姓名/手机号等敏感信息；

答案仅用于本次测评，不对外展示原始答题数据。

## 八、测评报告详细设计

### 8.1 报告整体框架

**核心理念：深度解读自我，精准定位爱情**

#### 第一部分：核心摘要 - 你的情感DNA

**个人情感画像**
- 五维度雷达图展示（情感安全感S、个人空间A、共同成长G、现实务实R、情绪表达E）
- 各维度分值与级别（极高90+、较高70-89、中等40-69、较低20-39、很低<20）
- 基于分值的个性化解读文案

**情感优势与挑战**
- 三大情感优势（基于用户得分最高的三个维度）
- 两大潜在挑战（基于可能冲突的维度组合）

#### 第二部分：深度维度解析

每个维度的详细分析：
- **S维度**：情感安全感需求
  - 日常相处中的6个行为特征
  - 压力状态下的反应模式
  - 给伴侣的5个安心行为建议
  - 5个绝对不能触碰的雷区

- **A维度**：个人空间需求
  - 个人空间需求的5个具体表现
  - 边界设定的4个方面

- **G维度**：共同成长重视度
  - 共同成长需求的4个具体表现

- **R维度**：现实务实程度
  - 现实需求的4个具体表现

- **E维度**：情绪表达偏好
  - 情感交流需求的4个具体表现

#### 第三部分：伴侣适配分析

**理想伴侣画像**
- 基于五维得分的核心特征要求
- 价值观匹配分析（生活观念、事业观念、家庭观念）
- 沟通风格匹配（表达方式、倾听习惯、冲突处理）

**八大伴侣类型适配度排名**
- 完整排名表格：适配度百分比 + 核心匹配点 + 潜在风险
- 适配度计算说明

**最适配类型深度解析**
- 类型定义和核心特征
- 这类人的具体特征（性格、价值观、情感表达、生活习惯）
- 五维度匹配分析（详细解释为什么适合）
- 和这类人在一起的生活场景（日常相处、情感交流、冲突处理、未来展望）
- 识别这类人的信号
  - 初识阶段的4个观察要点
  - 深入了解的4个验证方法

**避雷指南**
- 最不适合的类型分析
- 五维度冲突分析（详细的冲突原因和可能问题）
- 如果遇到这类人可能出现的问题（关系初期、长期相处、分手风险）

#### 第四部分：实战指导手册

**寻找期行动指南**
- 在哪里遇到最适合你的人（3个高频场所推荐）
- 初识阶段的筛选技巧
  - 前3次约会的观察要点
  - 深入了解的5个关键问题（含考察点、理想回答、红旗信号）

**深入了解期策略**
- 关系进展的节奏控制（4个阶段的时间线和目标）
- 推进关系的4个关键节点

**关系维护技巧**
- 基于用户情感特质的维系技巧（针对最高分和次高分维度）
- 适合用户性格的沟通模式
- 冲突处理技巧（预防、处理、修复）

### 8.2 报告生成逻辑

**动态内容匹配**
- 根据用户五维得分从文案库中匹配相应内容
- 基于得分组合生成个性化分析
- 适配度计算和排名

**文案库结构**
- 按维度分值区间预设文案模板

### 8.5 付费模式设计

**报告内容**

- 内容丰富，价值感强，完全支撑付费模式

**价值体现**
- 深度自我认知（约8000字的专业分析）
- 精准适配分析（科学计算+专业解读）
- 实用指导手册（可操作的行动建议）
- 长期参考价值（可反复使用的择偶指南）
